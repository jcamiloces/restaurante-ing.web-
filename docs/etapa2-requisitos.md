Etapa 2: Requisitos del Sistema

1. Actores del Sistema
Los actores representan los diferentes roles que interactúan con el software:

    Cliente: Usuario que interactúa con la interfaz digital para visualizar el menú, seleccionar productos y confirmar su orden.

    Sistema de Cocina (Receptor): Entidad lógica que recibe la información del pedido procesado para su preparación.

    Administrador del Proyecto: Responsable de supervisar que el sistema cumpla con los tiempos y metas establecidas (Líder y equipo de desarrollo)

2. Requisitos Funcionales (RF)

    RF1 - Visualización de Menú: El sistema debe mostrar de forma clara los productos disponibles, sus descripciones y precios.

    RF2 - Selección de Productos: El sistema debe permitir al usuario añadir múltiples productos a un carrito o lista de pedido temporal.

    RF3 - Cálculo de Total: El sistema debe calcular automáticamente el costo total de los productos seleccionados antes de confirmar la orden.

    RF4 - Confirmación de Pedido: El sistema debe capturar la intención de compra del usuario y simular el envío de la orden al área de preparación.

    RF5 - Interfaz Responsiva: El sistema debe proporcionar una interfaz digital funcional y fácil de usar para el cliente final.

3. Requisitos No Funcionales (RNF)

    RNF1 - Eficiencia (Tiempo): El sistema debe estar diseñado para reducir el tiempo de atención en al menos un 30% en comparación con métodos tradicionales.

    RNF2 - Usabilidad: La interfaz debe ser intuitiva, permitiendo que personas con conocimientos técnicos básicos puedan realizar un pedido sin asistencia.

    RNF3 - Precisión: El sistema debe garantizar que la información enviada a la cocina coincida exactamente con la selección del usuario para disminuir errores en los pedidos.

    RNF4 - Disponibilidad: Dado que es un prototipo académico, el sistema debe ser capaz de ejecutarse en entornos simulados de forma estable.

    RNF5 - Tecnologías de Soporte: El desarrollo debe apoyarse en herramientas estándar como Visual Studio y lógica de bases de datos para asegurar un entorno de trabajo colaborativo.
